# Device Identifier Collected but Undisclosed in Privacy Policy

## Problem Statement

The Terms of Use "Acceptance and Record-Keeping" section (line 159-164) states that when users accept the Terms, a "device identifier" is logged. However, the Privacy Policy's "Automatically Collected Information" section does NOT list "device identifier" among the data collected. This is a disclosure gap.

A device identifier is a much more sensitive data point than device type/OS version — it can uniquely identify a specific device and, by extension, a specific user.

## Findings

**Source:** Security Sentinel agent

**Evidence:**
- `TERMS_OF_USE.md` lines 159-164: "A device identifier" logged on acceptance
- `PRIVACY_POLICY.md` lines 21-23: Only lists "device type and operating system version"
- Privacy Policy IPP 13 mentions unique identifiers but doesn't list device identifier as collected data

**Impact:**
- Non-compliance with NZ IPP 1 (purpose of collection undisclosed)
- Non-compliance with APP 1 (open and transparent management)
- If GDPR applies: non-compliance with Article 13 (must disclose all data collected)

## Proposed Solutions

### Option A: Add device identifier to Privacy Policy disclosures (Recommended)
Add "device identifier" to the "Automatically Collected Information" section. Clarify:
- What type of identifier (UUID generated by the app? Apple IDFV? Something else?)
- Whether it is stored locally only or transmitted
- Retention period for acceptance logs

- **Pros:** Resolves disclosure gap, compliance
- **Cons:** None
- **Effort:** Small
- **Risk:** None

### Option B: Remove device identifier from acceptance logging
If the device identifier isn't strictly necessary, don't log it.

- **Pros:** Simplifies compliance, reduces data collected
- **Cons:** Loses acceptance audit trail completeness
- **Effort:** Small (legal doc change + app code change)
- **Risk:** Low
